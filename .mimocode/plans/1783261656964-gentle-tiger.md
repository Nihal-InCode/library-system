# Plan: Presentations Feature

## Goal
Add a "Presentations" screen accessible from the sidebar navigation where librarians can add, view, and delete presentation records. Each record stores a file (PDF/PPT), topic, presenter name, and date. Files are copied to a `presentations/` folder in the project. Clicking a presentation opens it with the system default viewer.

## 1. Database — `qt_ui/backend/db.py`

### Table schema (add in `_migrate()`)

```sql
CREATE TABLE IF NOT EXISTS presentations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    file_name TEXT NOT NULL,
    file_path TEXT NOT NULL,
    topic TEXT NOT NULL,
    presenter TEXT NOT NULL,
    event_date TEXT NOT NULL,
    added_date TEXT NOT NULL
)
```

- `file_name`: original filename (e.g., "AI_Presentation.pdf")
- `file_path`: stored path inside `presentations/` folder
- `topic`: can be long (multi-line)
- `presenter`: name of the person who presented
- `event_date`: date the presentation occurred (YYYY-MM-DD)
- `added_date`: when it was added to the app

### CRUD methods on `Database` class

- `add_presentation(file_name, file_path, topic, presenter, event_date)` → `(ok, message)`
- `get_presentations(page, page_size, search)` → `{"items": [...], "total": N, "hasMore": bool}`
- `delete_presentation(presentation_id)` → `(ok, message)`
- `get_presentation_file_path(presentation_id)` → `file_path` (for opening)

## 2. Bridge — `qt_ui/backend/app_bridge.py`

### New signals

```python
presentationsReady = Signal(str, arguments=['dataJson'])
presentationMutationResult = Signal(bool, str, arguments=['ok', 'message'])
```

### New slots

- `loadPresentations(page, pageSize, search)` — fetches paginated list, emits `presentationsReady`
- `addPresentation(filePath, topic, presenter, eventDate)` — copies file to `presentations/` folder, saves to DB, emits result
- `deletePresentation(presentationId)` — deletes from DB + disk, emits result
- `openPresentation(presentationId)` — opens file with `os.startfile()` (Windows) / `subprocess` (cross-platform)

### File copy logic in `addPresentation`

1. Resolve `presentations/` dir relative to project root (same logic as `images/`)
2. Create dir if missing
3. `shutil.copy2()` the source file into `presentations/`
4. Store the destination path in DB

## 3. QML Screen — `qt_ui/qml/screens/Presentations.qml` (NEW)

### Structure

```
Page
├── Header: "Presentations" title + "Add Presentation" button
├── Search bar (GlassTextField)
├── ListView of presentation cards
│   └── Each card: topic, presenter, date, file icon, open button, delete button
├── EmptyState (when no presentations)
├── Add Presentation Popup
│   ├── File selector (FileDialog + read-only GlassTextField + Browse button)
│   ├── Topic field (GlassTextField, multi-line via TextArea)
│   ├── Presenter name field (GlassTextField)
│   ├── Event date field (DatePicker or simple date input)
│   └── Save / Cancel buttons
└── Delete confirmation Dialog
```

### Key behaviors
- Files supported: PDF, PPT, PPTX, DOC, DOCX (via FileDialog nameFilters)
- Topic field uses `TextArea` (not `TextField`) to support multi-line input
- Date input: `TextField` with placeholder "YYYY-MM-DD" — simple and reliable
- Open button: calls `appBridge.openPresentation(id)`
- Cards show: topic (bold, up to 2 lines), presenter, date, file type badge
